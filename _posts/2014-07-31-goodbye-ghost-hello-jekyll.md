After a week of testing out [Ghost](http://tryghost.org) I decided it was too much work. I really liked Ghost but runnning it locally seemed to excessive. So I decided to ditch it for Jekyll which is recommneded by Github for Github Pages. 

I thought it would be simple because Github made it seem so. Unfortunately, I had some problems along the way and it was very time consuming process. My main problems revolved around _dependancy_ issues with _Ruby_. But after getting the correct dependancies installed I was flying. 

I also found a simple and elegant theme called Poole. You can try out the demo [here](http://demo.poole.com). Luckily, setting up Poole was easy though it was time consuming. But once it's setup all you have to do is start blogging. 

The main reasons why switched over were because of workflow and tools. My workflow under Ghost required many steps. I was able to cut this process in half with Jekyll! I was finally able to make the switch after I discovered a really cool _markdown_ editor called [Haroopad](http://pad.haroopress.com). 

With Haroopad I can easily write up my post without the need to fire up Ghost. And since Jekyll recognizes any markdown file in the _post_ directory as a "post" all I have to do is write my posts. No need to start the Ghost instance and log in and start writing. I just write my post in the correct format required by Jekyll, push to my repo and I'm done.

The biggest plus, is that my page is blazing fast. I haven't done a speed test yet but I can visually see the difference. I really like this new setup and I will be sticking to it. If anyone is looking to do this I highly recommend using Haroopad as your markdown editor.
